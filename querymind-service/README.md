# QueryMind Service

FastAPI backend service with Celery workers, RabbitMQ async history persistence, Redis caching, Alembic migrations, and AI-powered SQL generation.

## Architecture

```
Client / Gateway
      │  HTTP
      ▼
┌─────────────────────┐
│  FastAPI  :8001     │  ← request sanitization, request-ID middleware
│  /api/v1/           │
│    connections/add  │
│    query/ask        │──→ RabbitMQ ──→ Celery Worker ──→ PostgreSQL (history)
│    query/history    │
│    health/ready     │
└────────┬────────────┘
         │
    ┌────┴─────┐  ┌────────┐  ┌──────────┐
    │ Redis    │  │ Ollama │  │PostgreSQL│
    │ (cache)  │  │ (AI)   │  │ (store)  │
    └──────────┘  └────────┘  └──────────┘
```

## Non-blocking Query Flow

1. `POST /api/v1/query/ask` received  
2. Schema fetched from Redis cache (or introspected on miss)  
3. SQL generated by Ollama (→ HuggingFace fallback)  
4. SQL validated (SELECT-only, injection checks, table existence)  
5. SQL executed with pagination against user DB  
6. **Response returned immediately**  
7. History record published to RabbitMQ `history-persist-queue`  
8. `rabbitmq-consumer` reads message → fires `persist_history` Celery task  
9. Celery worker writes to `query_history` table in PostgreSQL  

## Running Locally

```bash
cp .env.example .env
# Edit ENCRYPTION_KEY and any other values

docker compose up -d
```

Migrations run automatically on service startup (`alembic upgrade head`).

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/v1/health` | Liveness probe |
| GET | `/api/v1/health/ready` | Readiness (DB + Redis + Ollama) |
| POST | `/api/v1/connections/add` | Store + introspect a DB connection |
| GET | `/api/v1/connections/{user_id}` | List connections for a user |
| DELETE | `/api/v1/connections/{connection_id}?user_id=` | Soft-delete a connection |
| POST | `/api/v1/query/ask` | Generate + execute SQL (non-blocking history) |
| POST | `/api/v1/query/schema/refresh` | Force schema re-introspection |
| GET | `/api/v1/query/history/{session_id}` | Retrieve query history |

## Session Setup

Before calling `/query/ask`, store the encrypted connection string in Redis with key `session:{session_id}:conn`. The gateway service is responsible for creating sessions and populating this key via the shared Redis instance.

## Running Migrations Manually

```bash
# Apply all migrations
alembic upgrade head

# Create a new migration (autogenerate from model changes)
alembic revision --autogenerate -m "add_column_xyz"

# Rollback one revision
alembic downgrade -1
```

## Running Celery Worker

```bash
celery -A app.workers.celery_app worker --loglevel=info --concurrency=4
```

## Running RabbitMQ Consumer

```bash
python rabbitmq_consumer.py
```
